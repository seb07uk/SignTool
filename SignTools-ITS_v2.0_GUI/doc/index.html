Code Signing GUI

Capabilities

Everything in one window

Complete code-signing workflow with auto-detection of tools, bilingual interface, and colour-coded real-time output.

🔒
Code Signing
Sign EXE, DLL, scripts using a certificate from the Windows store or a PFX file. Supports SHA-256, SHA-512, dual signing (/as) and debug mode.
⏱️
Timestamp
Add RFC3161 (modern, recommended) or Authenticode (legacy) timestamps to already-signed files. Configurable server and hash algorithm.
✅
Signature Verify
Verify all signatures (/pa) or page signatures (/pg) against optional catalog files. Colour-coded pass/fail output.
🗄️
CatDB & Remove
Add or remove catalog files from the Windows catalog database. Strip Authenticode signatures from binaries in bulk.
📋
Batch Mode
Built-in response file editor. Load, edit, and execute signtool batch jobs directly — one argument per line, first line is the command.
🔑
Certificate Generator
Generate self-signed, Root CA or CA-signed certificates via OpenSSL or PowerShell. RSA / EC keys, SAN, PFX export, Windows store install.
🌐
Bilingual UI
Full Polish / English interface switchable at runtime. No restart required. Built-in translation system with no external dependencies.
⚡
Auto-Detection
Scans all drives at startup for signtool.exe and openssl.exe. Summary dialog shows found paths; missing tools can be located manually.
🧵
Non-blocking Execution
All operations run in daemon threads. Output streams line-by-line into the GUI. No frozen window, no console flash on Windows.

Sign

Select one or more files for signing. Choose the certificate method — from the Windows certificate store (by subject name or SHA1 thumbprint) or from a PFX file with optional password.

Configure hash algorithm, description, URL, timestamp server type and hash, dual signing, and debug mode. Click 🔒 Sign or use Show Command to preview the exact signtool invocation.

/fd SHA256/n cert_name/sha1 /f pfx_file/p password/tr RFC3161 /t Authenticode/as dual_sign/debug

Timestamp / Verify

Add a timestamp to files that are already signed. Choose RFC3161 (modern, recommended for long-term validity) or Authenticode (legacy systems).

Verify signatures with /pa (all signatures) or /pg (page signature), with optional catalog file (/c). Results are colour-coded in the output panel.

/tr RFC3161/td SHA256/pa/pg/c catalog

CatDB / Remove

Manage the Windows catalog database: add or remove .cat catalog files using the catdb command. Requires administrator privileges.

The Remove section strips Authenticode signatures from binaries using signtool remove /s. Select files and click Remove Signatures.

catdb /addcatdb /delremove /s

Batch Mode

Load, edit, and execute signtool response files. Format: one argument per line, first line is the signtool command (sign, verify, timestamp, etc.).

Example response file:

sign
/n "My Certificate"
/fd SHA256
/tr http://timestamp.sectigo.com
/td SHA256
C:\Release\app.exe

🔑 Generate Certificate

Generate certificates via OpenSSL (cross-platform) or PowerShell New-SelfSignedCertificate (Windows only, exports to PFX).

Supported types: Self-signed, Root CA, Signed by CA. Configure: key algorithm (RSA / EC), key size, signature hash, validity, Subject fields (CN, O, C, L, ST, e-mail), SAN (DNS + IP), PFX export and Windows cert store installation.

Self-signedRoot CACA-signedRSA / ECSHA-256/384/512SANPFX export

Options

Switch the application language between Polish and English — no restart required. All UI labels update instantly.

Output

All operations stream their output here in real time. Lines are colour-coded: green for success, red for errors, blue for the executed command echo, and grey for separators.

The output widget is scrollable and retains history for the current session. Exit codes are displayed prominently at the end of each operation.

✓ success ✗ error cmd echo exit code

Getting Started

Install & Run

No dependencies. Download, run. Or compile to a portable EXE.

Run from source
# Clone the repository
git clone https://github.com/seb07uk/signtool-its-gui

# Run directly — no install needed
python main.py
Compile to EXE
# Install PyInstaller
pip install pyinstaller

# Build portable single-file EXE
pyinstaller --noconsole --onefile main.py
