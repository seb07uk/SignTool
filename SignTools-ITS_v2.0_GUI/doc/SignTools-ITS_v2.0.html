Overview
Przegląd

What is SignTool's-ITS GUI? Czym jest SignTool's-ITS GUI?

A professional, portable Windows application that wraps the full power of Microsoft signtool.exe and openssl.exe into an intuitive tabbed GUI. No command-line expertise required. Profesjonalna, przenośna aplikacja Windows, która udostępnia pełne możliwości Microsoft signtool.exe i openssl.exe w intuicyjnym interfejsie z zakładkami. Bez znajomości wiersza poleceń.

🔏
Microsoft SignTool

The official Windows SDK code-signing utility. Signs EXE, DLL, SYS, MSI, CAT, PS1 — any Authenticode-compatible file. Full timestamp, verify and CatDB support. Oficjalne narzędzie Windows SDK do podpisywania kodu. Podpisuje EXE, DLL, SYS, MSI, CAT, PS1 i każdy inny plik Authenticode. Pełna obsługa znacznikowania czasu, weryfikacji i CatDB.

🔑
OpenSSL

The cross-platform cryptographic toolkit. Used for generating self-signed, Root CA, and CA-signed certificates. Full RSA & EC key support, SAN extensions, PFX export. Wieloplatformowy zestaw narzędzi kryptograficznych. Służy do generowania certyfikatów self-signed, Root CA i podpisanych przez CA. Obsługa RSA i EC, rozszerzenia SAN, eksport PFX.

ℹ️
Target users: Software developers, IT administrators, DevOps engineers, Code Signing Officers, and security professionals who regularly sign binaries, drivers, installers, scripts, or catalog files on Windows. Docelowi użytkownicy: Deweloperzy, administratorzy IT, inżynierowie DevOps, oficerowie ds. podpisywania kodu i specjaliści bezpieczeństwa, którzy regularnie podpisują pliki binarne, sterowniki, instalatory, skrypty lub katalogi w systemie Windows.

Certificate Types & PurposeTypy i cel certyfikatów

SignTool's-ITS GUI supports three certificate types, each serving a distinct role in a PKI hierarchy. SignTool's-ITS GUI obsługuje trzy typy certyfikatów, każdy pełniący odrębną rolę w hierarchii PKI.

Self-SignedSamopodpisany

STANDALONEAUTONOMICZNY

A standalone certificate signed by its own private key. Not trusted by default in Windows without manual trust installation.Certyfikat podpisany własnym kluczem prywatnym. Domyślnie niezaufany w Windows bez ręcznej instalacji zaufania.

  • Use case: Development, testing, internal toolsZastosowanie: Środowisko dev, testy, narzędzia wewnętrzne
  • extendedKeyUsage=codeSigning
  • keyUsage=digitalSignature

Root CA

AUTHORITYURZĄD

A Certification Authority root certificate. Can sign other certificates. Forms the trust anchor of an internal PKI hierarchy.Certyfikat głównego urzędu certyfikacji. Może podpisywać inne certyfikaty. Stanowi kotwicę zaufania wewnętrznej hierarchii PKI.

  • Use case: Internal PKI, enterprise certificate hierarchyZastosowanie: Wewnętrzne PKI, korporacyjna hierarchia certyfikatów
  • basicConstraints=critical,CA:TRUE
  • keyUsage=keyCertSign,cRLSign

CA-SignedPodpisany przez CA

PRODUCTIONPRODUKCJA

An end-entity code-signing certificate issued by an internal Root CA. Trusted on all machines that trust the CA.Certyfikat końcowy do podpisywania kodu, wystawiony przez wewnętrzne Root CA. Zaufany na wszystkich maszynach, które ufają temu CA.

  • Use case: Production signing by internal CAZastosowanie: Produkcyjne podpisywanie przez wewnętrzne CA

RFC 3161 is the modern standard (flag: /tr). It supports SHA-256 and ensures the signature remains valid even after the signing certificate expires. Authenticode (flag: /t) is a legacy SHA1-only protocol compatible with Windows XP. Always use RFC 3161 unless you specifically need XP compatibility. RFC 3161 to nowoczesny standard (flaga: /tr). Obsługuje SHA-256 i zapewnia ważność podpisu nawet po wygaśnięciu certyfikatu. Authenticode (flaga: /t) to starszy protokół SHA1-only kompatybilny z Windows XP.
