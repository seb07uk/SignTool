🔍 The Problem Problem

On Windows, Python defaults to the system code page — cp1250, cp852, or cp1251 — depending on the locale. When a PyInstaller-packaged executable tries to print non-ASCII characters (Polish, Cyrillic, CJK), the output is corrupted or the app raises a UnicodeEncodeError and crashes.

A runtime hook solves this at the root: it runs before __main__ and all imports, setting UTF-8 before any character can be mishandled.

Na systemie Windows Python domyślnie używa strony kodowej systemu — cp1250, cp852 lub cp1251 — w zależności od ustawień regionalnych. Gdy plik wykonywalny spakowany przez PyInstaller próbuje wypisać znaki spoza ASCII (polskie, cyrylicę, CJK), wynik jest zniekształcony lub aplikacja rzuca UnicodeEncodeError i crashuje.

Hook startowy rozwiązuje to u źródła: uruchamia się przed __main__ i wszystkimi importami, ustawiając UTF-8 zanim jakikolwiek znak może zostać błędnie obsłużony.

🐛 Bugs Fixed Naprawione Błędy

HIGH sys.stdin not reconfigured sys.stdin nie był rekonfigurowany

Reading UTF-8 piped input could raise UnicodeDecodeError or produce silent corruption. Fixed: added _reconfigure(sys.stdin, ...). Odczyt potokowego wejścia UTF-8 mógł rzucić UnicodeDecodeError lub produkować ciche uszkodzenia. Naprawa: dodano _reconfigure(sys.stdin, ...).

MED stderr used errors='replace' stderr używał errors='replace'

Replacing undecodable diagnostic bytes with '?' destroys forensic information. Fixed: stderr now uses errors='backslashreplace'. Zastępowanie nierozkodowalnych bajtów diagnostycznych znakiem '?' niszczy informacje potrzebne do debugowania. Naprawa: stderr używa teraz errors='backslashreplace'.

MED Silent exception suppression Ciche tłumienie wyjątków

except Exception: pass — failures were invisible. Fixed: now emits a diagnostic line to stderr so developers know the hook failed. except Exception: pass — błędy były niewidoczne. Naprawa: teraz emituje linię diagnostyczną do stderr, by deweloperzy wiedzieli, że hook nie zadziałał.

LOW Weak type guard (hasattr only) Słabe sprawdzenie typu (tylko hasattr)

Custom stream wrappers could have a reconfigure attribute with different semantics. Fixed: added isinstance(io.TextIOWrapper) check. Niestandardowe wrappery strumieni mogły mieć atrybut reconfigure o innej semantyce. Naprawa: dodano sprawdzenie isinstance(io.TextIOWrapper).

LOW PYTHONUTF8 set unconditionally PYTHONUTF8 ustawiany bezwarunkowo

PYTHONUTF8 requires Python ≥ 3.7. Fixed: guarded by sys.version_info >= (3, 7), with a warnings.warn on older versions. PYTHONUTF8 wymaga Pythona ≥ 3.7. Naprawa: zabezpieczono przez sys.version_info >= (3, 7), ze warnings.warn dla starszych wersji.

INFO Missing __all__ declaration Brak deklaracji __all__

PyInstaller's hook scanner could accidentally pick up module-level names. Fixed: added __all__: list = []. Skaner hooków PyInstaller mógł przypadkowo pobrać nazwy na poziomie modułu. Naprawa: dodano __all__: list = [].

🚀 Installation Instalacja

1
Copy the file to your project Skopiuj plik do projektu 
myproject/
├── myapp.py
├── myapp.spec
└── hooks/
    └── hook_utf8.py
2
Add to your .spec file Dodaj do pliku .spec 
# myapp.spec
a = Analysis(
    ['myapp.py'],
    ...
    runtime_hooks=['hooks/hook_utf8.py'],
    ...
)
3
Or use the CLI Lub użyj linii poleceń 
pyinstaller myapp.py \
    --runtime-hook hooks/hook_utf8.py \
    --onefile
