Wszystko czego
potrzebujesz.

🔍
Auto-wykrywanie signtool
Przeszukuje zasoby PyInstaller, katalog programu, wszystkie ścieżki Windows SDK 8.0–11+ (x64, x86, arm64, arm) i PATH systemu. Żadnej ręcznej konfiguracji.
📂
Auto-wykrywanie certyfikatów
Skanuje 9 lokalizacji Windows w określonej kolejności — od bundla PyInstaller po katalog certs/ i profil użytkownika. Deduplicacja przez realpath().
⚡
Asynchroniczne podpisywanie
Proces signtool.exe działa w wątku daemon. UI nigdy się nie blokuje. Wyniki przekazywane thread-safe przez root.after().
🕐
RFC 3161 Timestamp
Znacznik czasu dodawany automatycznie przez https://timestamp.digicert.com z algorytmem SHA-256. Podpis ważny po wygaśnięciu certyfikatu.
📦
PyInstaller ready
Pełna obsługa trybu frozen. Zasoby wczytywane z sys._MEIPASS. Kompilacja do standalone .exe jedną komendą — bez żadnych zmian w kodzie.
📋
Dziennik operacji
Każde podpisanie rejestrowane w sign.log — data, ścieżka pliku, certyfikat, wynik. Pełna diagnostyka błędów signtool w STDOUT i STDERR.

Siedem kroków
do bezpiecznego podpisu.

01
Uruchom aplikację
python main.py lub QuickDigitalSignature.exe — bez instalacji SDK.
02
Wybierz plik
Browse... → .exe, .dll, .msi, .cab lub dowolny inny plik PE.
03
Certyfikat auto-wykryty
Aplikacja automatycznie znajduje .pfx z katalogu certs/ lub profilu Windows.
04
Kliknij Sign File
Walidacja wejść, budowa argumentów, start wątku daemon.
05
Podpisywanie w tle
signtool.exe sign /fd sha256 /tr https://... — UI pozostaje responsywny.
06
Znacznik czasu RFC 3161
Połączenie z timestamp.digicert.com, zapis znacznika SHA-256 w podpisie.
07
Wynik w głównym wątku
root.after() przekazuje sukces lub błąd — pełny log w sign.log.

Zaprojektowane
z myślą o bezpieczeństwie.

Każdy aspekt podpisywania zaprojektowano minimalizując obszar ataku i ekspozycję wrażliwych danych.

🛡️
shell=False · brak injection
subprocess.run() z listą argumentów. Zero ryzyka command injection nawet z niebezpiecznymi ścieżkami plików.
🔐
Hasło zerowane natychmiast
Slot /p w liście cmd jest nadpisywany pustym stringiem w bloku finally — minimalna ekspozycja hasła PFX w pamięci.
🔒
HTTPS timestamp
Serwer DigiCert połączony przez TLS (https://). Poprzednia wersja używała HTTP — poprawione w v1.1.0.
✅
Walidacja każdego wejścia
os.path.isfile() przed każdą operacją. Jasne komunikaty o błędach bez ujawniania ścieżek systemowych.
